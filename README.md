# 📊 交易技巧 002 - 信号驱动过滤系统 (v2.1)

## ⚠️ 【重要】v2.1 重大更新说明 (2026-03-27)

> **"收益率是市场给的反馈，不是预设的目标"**

| 问题 | v2.0 的错误理解 | v2.1 的正确理解 |
|------|--------------|---------------|
| **为什么要设止盈止损？** | "我觉得涨/跌到某个点就该卖" | ❌ 这是在教市场怎么走势 |
| **什么时候卖出？** | 到价格点 or MACD 死叉 | ✅ **买入信号消失即卖** |
| **收益太低怎么办？** | "调低止盈点" | ✅ **收紧入场标准** |
| **收益率是谁决定的？** | 我设定的目标值 | ✅ **市场配合我的筛选标准后的结果** |

**核心理念**：让利润奔跑，让市场决定上限，只用信号反转作为退出条件。

## 📋 核心配置

### 固定参数 (不可变)

| 参数 | 值 | 说明 |
|------|-----|------|
| MACD Short | 12 | 短期均线 |
| MACD Long | 26 | 长期均线 |
| MACD Signal | 9 | 信号线 |
| 退出条件 | 信号反转 | 无预设止盈止损 |

### 硬约束门槛 (Pass/Fail)

| 约束 | 阈值 | 作用 |
|------|------|------|
| 总收益率 | ≥ 10% | 收益过滤 |
| 夏普比率 | ≥ 0.5 | 风险控制 |
| 最大回撤 | ≤ 25% | 回撤控制 |
| 盈亏比 | ≥ 1.5 | 质量要求 |

## 💡 使用原则

### ✅ 正确做法

```
1. 随机抽样验证 → 找出符合我系统的标的
2. 只对通过的标的执行交易
3. 大部分时间空仓等待
4. 定期重新验证（市场会变）
5. 如果平均收益太低，收紧入场标准而非调止盈点
```

### ❌ 错误做法

```
1. ❌ 为每只股票单独调参 → 过拟合
2. ❌ 追求高通过率 → 放弃纪律
3. ❌ 因为某只股票"看起来好"就建仓 → 不守规则
4. ❌ 认为空仓=错过机会 → 空仓也是持仓
5. ❌ 预设止盈止损 → 告诉市场只能赚多少
```

## ⚠️ 风险提示

### 预期管理

- **通过率会很低**：10% 以下是正常的，这是好的纪律
- **不要修改参数提高通过率**：那是过拟合的信号
- **空仓是常态**：等待符合条件的标的是工作的一部分

### 风险警告

```
⚠️ 历史回测 ≠ 未来表现
⚠️ 过拟合风险：持续在更多随机样本上验证（建议≥50 只）
⚠️ 实盘可能低于回测：留足安全边际
⚠️ 单一策略风险：不要 All-in，分散投资
⚠️ 信号延迟：MACD 指标天然滞后，需注意滑点成本
```

### 数据依赖

当前版本依赖腾讯财经 API 获取历史数据：
- ✅ 实时价格：可用
- ⚠️ 历史 K 线：需要 `tencent_stock_api.py` 支持
- ❌ 无数据时：使用模拟数据演示逻辑
